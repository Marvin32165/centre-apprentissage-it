Le modèle OSI

Sept couches qui décrivent comment deux machines communiquent, de l'application jusqu'au câble. Chaque couche rend service à celle du dessus et s'appuie sur celle du dessous — c'est la grille de lecture de tout problème réseau.

02

Configuration de base & sécurité

On accède au routeur par le câble console (rollover) branché sur le port RS-232 du PC. La configuration suit toujours la même séquence : nom, mots de passe, bannière, lignes d'accès, puis chiffrement et SSH.

Les modes de l'IOS

  • Router> — mode utilisateur (lecture seule).
  • Router# — mode privilégié (enable).
  • Router(config)# — mode configuration globale (configure terminal).

Séquence de base + sécurité

Cisco IOS — R1
enable
configure terminal
 hostname R1
 no ip domain-lookup              ! évite les recherches DNS sur fautes de frappe
 banner motd #Admin seulement#

 enable secret ciscocisco         ! mot de passe privilégié (chiffré)
 service password-encryption       ! chiffre tous les mots de passe en clair
 security passwords min-length 10

 line console 0                    ! accès par le câble console
  password cisco
  login
  exec-timeout 5
 exit

Activer SSH (et bannir Telnet)

Cisco IOS — SSH
 ip domain name cisco.lan
 username admin privilege 15 secret motdepasse
 crypto key generate rsa             ! choisir 1024 ou 2048 bits
 ip ssh version 2

 line vty 0 15                       ! lignes d'accès distant
  login local                       ! authentifie via les comptes locaux
  transport input ssh               ! SSH uniquement, pas Telnet
 exit
end
write                                 ! enregistre la running-config

Réflexes utiles

do show running-config (ou do sh run) pour voir la config sans quitter le mode config ; do write (ou do wr) pour sauvegarder. enable secret est chiffré, enable password ne l'est pas — toujours préférer secret.

03

VLAN & trunk

Un VLAN segmente un switch physique en plusieurs réseaux logiques isolés. Un port access appartient à un seul VLAN (un PC) ; un port trunk transporte tous les VLAN entre équipements grâce au marquage 802.1Q (dot1q).

Cisco IOS — SW1
enable
configure terminal
 hostname SW1

 ! Déclarer les VLAN
 vlan 10
  name WIFIGUEST_V10
 vlan 20
  name SERVERS_V20

 ! Port trunk vers le routeur / un autre switch
 interface g0/1
  switchport mode trunk
  switchport trunk allowed vlan all
  no shutdown

 ! Port access vers un PC du VLAN 10
 interface fa0/2
  switchport mode access
  switchport access vlan 10
  spanning-tree portfast        ! port d'extrémité : pas de calcul STP
  no shutdown

À retenir

Access = un VLAN, un terminal. Trunk = tous les VLAN, marqués en dot1q, entre switches/routeurs. Sans trunk, deux PC du même VLAN sur deux switches différents ne se parleraient pas.

04

Routage inter-VLAN (router-on-a-stick)

Les VLAN sont isolés par conception : pour les faire communiquer, il faut un routeur. La méthode router-on-a-stick utilise une seule interface physique découpée en sous-interfaces, une par VLAN, chacune servant de passerelle.

Cisco IOS — R1 (g0/0/0 = routeur/module/port)
interface g0/0/0
 no shutdown                       ! l'interface physique reste sans IP

! Une sous-interface = une passerelle de VLAN
interface g0/0/0.10
 encapsulation dot1q 10            ! associe la sous-interface au VLAN 10
 ip address 172.16.0.1 255.255.192.0
 no shutdown

interface g0/0/0.20
 encapsulation dot1q 20
 ip address 172.16.64.1 255.255.240.0
 no shutdown

À retenir

La passerelle d'un VLAN = l'IP de sa sous-interface sur le routeur. La notation g0/0/0 se lit routeur / module / port ; le .10 est la sous-interface, liée au VLAN par encapsulation dot1q 10. Sur un switch L3, on remplacerait cela par des interfaces vlan X (SVI).

05

DHCP sur Cisco

Le routeur peut jouer le serveur DHCP de chaque VLAN : on définit un pool par réseau, on exclut les adresses fixes (passerelle, serveurs) et on fournit passerelle, DNS et nom de domaine.

Cisco IOS — pool DHCP
! Réserver les adresses qui ne doivent PAS être distribuées
ip dhcp excluded-address 172.16.0.1 172.16.0.20

ip dhcp pool WIFI-GUEST_V10
 network 172.16.0.0 255.255.192.0
 default-router 172.16.0.1         ! la passerelle = sous-interface du VLAN
 dns-server 172.16.64.11
 domain-name wifiguest.vlan10.lan
exit

Piège classique

Toujours exclure la passerelle et les adresses statiques avant de définir le pool, sinon le routeur risque de les attribuer à un client et de créer un conflit d'adresses IP.

06

Subnetting & VLSM

Le VLSM (Variable Length Subnet Mask) découpe un grand bloc en sous-réseaux de tailles différentes, ajustées au nombre d'hôtes de chaque VLAN — sans gaspiller d'adresses.

La règle de calcul

  • Un préfixe /n laisse 32 − n bits d'hôte → 2^(32−n) − 2 adresses utilisables (on retire l'adresse réseau et le broadcast).
  • On classe les VLAN du plus grand au plus petit, puis on attribue les blocs dans l'ordre pour éviter les chevauchements.
PréfixeMasqueHôtes utiles
/24255.255.255.0254
/22255.255.252.01022
/20255.255.240.04094
/18255.255.192.016382
/27255.255.255.22430

Exemple de plan (bloc 192.168.0.0/17)

Plan VLSM par VLAN
VLAN 10  Wifi_guest   192.168.0.0/18    (16382 hôtes)
VLAN 20  Servers      192.168.64.0/20   (4094 hôtes)
VLAN 30  Hosts        192.168.80.0/20   (4094 hôtes)
VLAN 40  Management   192.168.96.0/22   (1022 hôtes)
VLAN 50  Wifi         192.168.100.0/22  (1022 hôtes)
VLAN 60  IoT          192.168.104.0/24  (254 hôtes)
VLAN 70  Printers     192.168.105.0/27  (30 hôtes)

À retenir

VLSM = la bonne taille pour chaque réseau. On commence par le VLAN le plus peuplé pour réserver les grands blocs, puis on descend. Méthode infaillible : trier par nombre d'hôtes décroissant et avancer séquentiellement dans l'espace d'adressage.

07

Routage statique & dynamique

Un routeur ne connaît au départ que ses réseaux directement connectés. Pour atteindre les autres, on lui indique les routes — soit à la main (statique), soit via un protocole qui les apprend tout seul (dynamique).

Route statique

Cisco IOS
! Route précise : pour atteindre 10.2.0.0/18, passer par 10.0.0.2
ip route 10.2.0.0 255.255.192.0 10.0.0.2

! Route par défaut : tout le reste part vers la passerelle
ip route 0.0.0.0 0.0.0.0 10.0.0.2

Les protocoles dynamiques rencontrés

ProtocoleTypeIdée clé
RIP v2Vecteur de distanceSimple, métrique = nombre de sauts (max 15)
OSPFÉtat de lienCalcule le plus court chemin (coût), notion d'aires
EIGRPHybride (Cisco)Convergence rapide, métrique composite
BGPVecteur de cheminRoutage entre systèmes autonomes (Internet)
Cisco IOS — OSPF mono-aire (exemple)
router ospf 1
 network 10.2.0.0 0.0.63.255 area 0
 network 10.0.0.0 0.0.0.3 area 0

À retenir

Statique : maîtrise totale, mais à maintenir à la main → parfait pour les petits réseaux et la route par défaut. Dynamique (OSPF, EIGRP) : les routeurs s'échangent leurs routes et s'adaptent aux pannes → indispensable dès que le réseau grandit. FHRP (HSRP) ajoute une passerelle virtuelle redondante.

08

NAT & ACL

Deux briques de bordure : le NAT traduit les adresses privées en adresses publiques pour sortir sur Internet ; les ACL filtrent le trafic selon des règles autorisant ou bloquant des flux.

NAT — trois saveurs

  • NAT statique — une privée ↔ une publique, fixe (ex. publier un serveur).
  • NAT dynamique — un pool de publiques partagé à la demande.
  • PAT (overload) — toutes les privées derrière une seule publique, distinguées par le port. C'est le NAT « de la box ».
Cisco IOS — PAT
access-list 1 permit 192.168.0.0 0.0.255.255
ip nat inside source list 1 interface g0/0/1 overload

interface g0/0/0
 ip nat inside
interface g0/0/1
 ip nat outside

ACL — filtrer le trafic

Une ACL est une liste ordonnée de règles permit/deny évaluée de haut en bas ; la première qui correspond s'applique, avec un deny any implicite à la fin.

Cisco IOS — port-security + ACL
! Port-security : limiter les MAC sur un port d'accès
interface fa0/2
 switchport port-security
 switchport port-security maximum 2
 switchport port-security violation shutdown

! ACL standard : autoriser seulement le VLAN management vers vty
access-list 10 permit 10.2.104.0 0.0.7.255
line vty 0 15
 access-class 10 in

À retenir

NAT/PAT = sortir sur Internet en économisant les adresses publiques. ACL = pare-feu basique, évalué dans l'ordre avec un deny implicite final — d'où l'importance de l'ordre des règles. Port-security verrouille un port d'accès sur les MAC autorisées.

Exercice final — la topologie d'entreprise

Le capstone réunit tout : un plan VLSM sur 10.2.0.0, une douzaine de VLAN dimensionnés, le DHCP par VLAN, la sécurité (SSH, mots de passe), le routage inter-VLAN et la sortie Internet. C'est l'exercice de synthèse, à refaire de zéro dans Packet Tracer pour valider le module.

La méthode en 6 temps

  1. 01
    Plan VLSMTrier les VLAN par nombre d'hôtes, attribuer les blocs sans chevauchement.
  2. 02
    Sécuriser les équipementsHostname, enable secret, service password-encryption, SSHv2, login local.
  3. 03
    VLAN & trunksCréer les VLAN sur les switches, ports access pour les terminaux, trunks entre équipements.
  4. 04
    Routage inter-VLANSous-interfaces dot1q (router-on-a-stick) ou SVI sur switch L3, une passerelle par VLAN.
  5. 05
    DHCP par VLANUn pool par réseau, exclusions des passerelles, DNS & domaine.
  6. 06
    Sortie & filtrageRoute par défaut, NAT/PAT vers Internet, ACL pour cloisonner les VLAN sensibles.

À retenir

Un projet réseau complet s'enchaîne toujours dans cet ordre : adressage → sécurité → VLAN → routage → services (DHCP/DNS) → bordure (NAT/ACL). Maîtriser cette séquence, c'est tenir l'examen pratique CCNA.
