01

Linux & le terminal

Avant de taper la moindre commande, comprenons de quoi on parle.

Linux est un système d'exploitation, comme Windows ou macOS. Debian est une « distribution » Linux : un ensemble complet, stable et gratuit, très utilisé sur les serveurs. Le terminal (ou shell) est l'outil principal pour l'administrer : on tape des commandes en texte au lieu de cliquer — plus rapide, plus précis, et ça fonctionne même sans interface graphique.

Anatomie d'une commande

Bash
ls -la /home   # commande + options + cible
  • ls — le nom de la commande (ici : lister des fichiers) ;
  • -la — des options qui modifient son comportement ;
  • /home — l'argument : sur quoi la commande agit.

Les versions de Debian

Nom de codeVersion
bullseyeDebian 11
bookwormDebian 12
trixieDebian 13
stable / testing / unstablestable actuelle / future / à éviter en production

À retenir

Pour connaître sa version : lsb_release -a ou cat /etc/os-release. Le nom de code (ex. trixie) sert partout, notamment dans la configuration des dépôts APT.

02

Premiers pas post-install

Debian vient d'être installé. Voici les toutes premières actions : créer son utilisateur et lui donner les droits sudo.

À la première connexion, on entre avec le compte root — le « super-utilisateur » qui a tous les droits. Travailler en permanence avec est dangereux : une commande mal tapée peut détruire le système. La bonne pratique : un utilisateur normal + sudo pour élever ses droits ponctuellement.

1
Installer sudo

Connecté en root : apt install sudo

2
Créer l'utilisateur

adduser apprenant (si pas déjà fait pendant l'installation).

3
L'ajouter au groupe sudo

usermod -aG sudo apprenant puis vérifier avec groups apprenant — « sudo » doit apparaître.

4
Se reconnecter

exit pour quitter root, puis se reconnecter avec le nouvel utilisateur.

Bash — en root
apt install sudo
adduser apprenant
usermod -aG sudo apprenant
groups apprenant        # vérifier que "sudo" apparaît
exit

Vérifications de base

Bash
lsblk                    # disques et partitions
ip a                     # adresses IP
apt list --upgradable    # mises à jour disponibles

À retenir

Ensuite, on préfixe les commandes d'administration par sudo : par exemple sudo apt update. Le système demande votre mot de passe, pas celui de root.

06

Liens & inodes

Comprendre l'inode explique pourquoi mv est instantané et ce qui distingue vraiment un lien physique d'un raccourci.

Chaque fichier est décrit par une structure appelée inode : taille, propriétaire, permissions, dates, et les adresses des blocs de données sur le disque. Le nom du fichier, lui, n'est qu'une entrée dans un dossier qui pointe vers un inode — un même inode peut avoir plusieurs noms.

Bash
ls -li fichier      # -i affiche le numéro d'inode en 1re colonne

Liens physiques (hard links)

Un lien physique fait pointer un second nom vers le même inode. Modifier l'un modifie l'autre : c'est littéralement le même fichier.

Bash
ln fichier1 fichier4
ls -li               # même inode pour fichier1 et fichier4, compteur de liens = 2
rm fichier1          # fichier4 reste intact : le contenu n'est libéré que si TOUS les liens disparaissent

Limite

Impossible de créer un lien physique vers un dossier. Pour un dossier, on utilise un lien symbolique.

Liens symboliques (raccourcis)

Un lien symbolique (l en 1er caractère dans ls -l) est un fichier à part qui contient juste le chemin vers la cible — comme un raccourci Windows.

Bash
ln -s fichier4 fichier5
ls -li               # inode DIFFÉRENT de fichier4, colonne des droits commence par "l"

Lien cassé

Si la cible d'un lien symbolique est supprimée, le lien devient « orphelin » (souvent affiché en rouge/clignotant par ls) : il pointe vers un chemin qui n'existe plus.

Le triplet des droits : u, g, o

Les droits Linux sont définis à l'aide de 3 bits (r/w/x) répétés pour 3 catégories d'utilisateur (u/g/o). Exemple avec rwxr-x--- — chaque case ci-dessous est reliée à ce qu'elle peut contenir :

u — propriétaire (user)
r
lectureposition 2 — r (présent) ou - (absent)
w
écritureposition 1 — w ou -
x
exécutionposition 0 — x ou -
g — groupe (group)
r
lecturemême règle, pour le groupe propriétaire du fichier
-
écriture- ici : le groupe ne peut pas écrire
x
exécutionle groupe peut exécuter (pour un dossier : « traverser »)
o — autres (others)
-
lectureabsente : personne d'autre ne peut lire
-
écrituretoujours absente pour « others » ici
-
exécutionaccès nul pour tous les autres

chmod accepte aussi une notation symbolique, relative aux droits déjà en place :

Bash — chmod symbolique
chmod u+rwx fichier   # ajoute lecture+écriture+exécution au propriétaire
chmod g+w fichier     # ajoute l'écriture au groupe
chmod o-r fichier     # retire la lecture aux autres
chmod u=rwx,g=rx,o=r fichier   # fixe les droits (pas relatif) : équivaut à 754

Cas pratique : dossier partagé en équipe

Pour qu'un groupe puisse déposer des fichiers dans un dossier commun sans casser les droits du propriétaire : chmod g+w dossier_partage ajoute l'écriture au groupe en gardant intacts les droits de u et o (contrairement à chmod 775 qui réécrit tout le triplet).

adduser vs useradd

adduser/deluser/addgroup/delgroup sont spécifiques à Debian (scripts conviviaux, interactifs). Les commandes Unix universelles useradd/userdel/groupadd/groupdel existent partout mais sont plus basiques (avec useradd, il faut lancer passwd soi-même pour activer le compte).

umask & ACL

umask définit quels droits sont retirés par défaut à la création d'un fichier ou dossier. Valeur typique : 0022, qui donne des fichiers en 644 et des dossiers en 755 (jamais l'exécution sur un fichier créé « normalement »). Elle se règle dans /etc/profile, /etc/login.defs, /etc/bash.bashrc ou ~/.bashrc. Les ACL vont plus loin en donnant des droits à un utilisateur ou groupe précis :

Bash — ACL
sudo apt install acl
getfacl dossier                    # afficher les ACL en place
setfacl -m g:sav:rwx dossier       # donner rwx au groupe "sav" sur ce dossier

08

Redirections & pipes

La vraie puissance du terminal : brancher les commandes entre elles.

SymboleRôle
>écrit la sortie dans un fichier (écrase l'existant)
>>écrit la sortie dans un fichier (ajoute à la fin)
<donne un fichier en entrée à une commande
|envoie la sortie d'une commande vers une autre (le « pipe »)
&&exécute la 2ᵉ commande seulement si la 1ʳᵉ réussit
||exécute la 2ᵉ commande seulement si la 1ʳᵉ échoue
Bash — exemples concrets
ls > liste.txt               # écrit le résultat dans liste.txt (écrase)
echo "log" >> liste.txt      # ajoute une ligne à la fin
wc -l < liste.txt            # compte les lignes du fichier
cat liste.txt | grep "log"   # cherche "log" dans le contenu
sudo apt update && sudo apt upgrade   # upgrade SI update a réussi

À retenir

echo $? affiche le code de sortie de la dernière commande : 0 = succès, autre chose = erreur. C'est ce que && et || utilisent pour décider.

09

sort, cut, wc, tee — flux & fichiers structurés

Quatre outils simples, chacun avec un rôle précis, qui se combinent par pipe pour traiter du texte structuré (CSV, listes, logs) sans écrire de script.

wc, sort, cut : les options clés

OutilOptions utiles
wc-l lignes · -w mots · -c octets · -L longueur de la plus longue ligne
sort-r décroissant · -n numérique · -k N champ N · -u sans doublons · -h tailles 1K/2M · -o vers fichier
cut-c 1-5 caractères · -f 1,3 champs · -d',' délimiteur · --complement tout sauf
Bash
ls /etc | wc -l                              # nombre de fichiers dans /etc

tee — afficher ET sauvegarder en même temps

tee lit l'entrée standard et l'écrit à la fois vers un fichier et vers l'écran. Utile pour écrire dans un fichier protégé avec sudo : sudo echo "x" > /etc/fichier échoue car la redirection > est interprétée par le shell AVANT que sudo s'applique.

Bash
echo "config" | sudo tee /etc/fichier            # écrit avec les droits root, affiche aussi
echo "config" | sudo tee /etc/fichier > /dev/null   # idem, sans afficher
date | tee -a journal.log                        # -a : ajoute au lieu d'écraser

Exercice guidé : grep, cut, uniq, sort et tubes

Un carnet d'adresses telephone.txt (12 personnes, format nom<TAB>prénom<TAB>numéro) sert de terrain d'entraînement classique pour enchaîner ces commandes.

QuestionCommande
Répertoires de connexion des utilisateurscut -d: -f6 /etc/passwd
Nombre de shells différentscut -d: -f7 /etc/passwd | sort | uniq | wc -l
Carnet trié par ordre alphabétiquesort telephone.txt
Nombre de personnes dans le carnetwc -l telephone.txt
Toutes les lignes des « Dupond »grep "Dupond" telephone.txt
Toutes les lignes hors « Dupond »grep -v "Dupond" telephone.txt
Téléphone du 1er « Dupond » du fichiergrep "Dupond" telephone.txt | head -1 | cut -f3
Téléphone du 1er « Dupond » par ordre alpha de prénomgrep "Dupond" telephone.txt | sort | head -1 | cut -f3

Pour la dernière ligne : grep garde les 4 Dupond, sort les trie — comme « Dupond » est identique pour tous, le tri se fait de fait sur le 2e champ (le prénom) — puis head -1 garde le premier (Agnès) et cut -f3 extrait son numéro.

Piège : wc -l vs grep -c ""

Si la dernière ligne du fichier n'a pas de retour à la ligne final, wc -l sous-compte d'une unité (11 au lieu de 12). grep -c "" telephone.txt compte les lignes réelles peu importe la fin de fichier — un réflexe plus fiable.

Piège d'encodage rencontré en pratique

Un fichier transféré depuis Windows peut être en ISO-8859-1 au lieu d'UTF-8 : les accents (« Agnès ») posent alors problème à grep (qui peut signaler « binary file matches »). Diagnostic : file telephone.txt · correction : iconv -f ISO-8859-1 -t UTF-8 telephone.txt -o telephone.utf8.txt.

10

sed, awk & expressions régulières

Transformer un flux de texte à la volée, sans ouvrir d'éditeur — indispensable pour scripter des traitements.

Les méta-caractères de base

SymboleSignificationExemple
.n'importe quel caractèreS.B → SEB, SAB, SSB…
?0 ou 1 foisS.?B → SB ou SEB
*0 ou n foisS.*B → SB, SEB, SAEIOUB…
+1 ou n fois
^ / $début / fin de ligne
[a-z]un caractère parmi cet ensemble[^abc] = tout SAUF a, b, c

sed — Stream EDitor

Bash
sed 's/Debian/Ubuntu/' /etc/os-release   # remplace la 1re occurrence par ligne, affiche seulement
sed -i 's/Debian/Ubuntu/' fichier        # -i modifie le fichier directement (in place)
sed -n 2,6p /etc/passwd                  # affiche uniquement les lignes 2 à 6

awk — traitement par colonnes

Bash
awk '{gsub(/.ebian/,"Ubuntu")}1' /etc/os-release   # cherche et remplace
awk '!/.ebian/' /etc/os-release                    # supprime les lignes qui matchent
awk 'NR==2,NR==4' /etc/os-release                  # affiche seulement les lignes 2 à 4

Enchaîner grep, sort, cut avec le pipe

Bash
cat /etc/passwd | grep nologin | sort | cut -d: -f1
# filtre les comptes "nologin", trie, ne garde que le login (1er champ)

find — rechercher des fichiers avec précision

find parcourt une arborescence et filtre par nom, type, taille, date ou permissions — et peut agir directement sur les résultats trouvés.

Syntaxe
find [chemin] [critères] [action]
CritèreEffet

-type f / d / lfichiers / dossiers / liens symboliques

Combiner des critères et agir

ActionEffet
-exec cmd {} \;exécute cmd sur chaque résultat

Bash
find / -type f \( -name "*.log" -or -name "*.txt" \)   # OU logique
find /tmp -name "*.tmp" -exec rm {} \;              # supprimer tous les .tmp

-exec vs -delete

-exec ... rm {} \; lance une commande rm par fichier trouvé (plus lent mais flexible) ; -delete est l'option native de find, plus rapide pour une simple suppression.

Variables & personnalisation du prompt

Le shell garde des informations en mémoire dans des variables — dont certaines pilotent l'apparence même du terminal.

Créer, exporter, afficher une variable

Bash
prenom="jean-denis"
echo $prenom                  # affiche la valeur
export formation="Linux"     # devient une variable d'environnement
env | grep formation         # maintenant visible par les autres processus
unset formation              # supprime la variable (session en cours)

Variable shell ≠ variable d'environnement

Une variable créée simplement (formation=Linux) n'existe que dans le shell courant : invisible pour les autres processus (absente de env). Il faut l'exporter. Pour qu'elle survive à la fermeture du terminal, ajouter la ligne export dans ~/.bashrc (un utilisateur) ou un .sh sous /etc/profile.d/ (tous), puis source ~/.bashrc.

echo et les codes d'échappement

echo n'interprète les codes d'échappement (comme \n) qu'avec l'option -e : echo -e "Bonjour\nÀ bientôt". Codes utiles : \n retour à la ligne · \t tabulation · \\ backslash littéral.

Personnaliser le prompt avec PS1

La variable $PS1 définit l'apparence de l'invite de commande.

CodeSignification
\unom d'utilisateur
\hnom d'hôte (court)
\wrépertoire courant (~ pour le home)
\$$ pour un utilisateur normal, # pour root
\ddate
Bash
PS1='\u@\h:\w\$ '                 # exemple de prompt personnalisé
echo "PS1='\u@\h:\w\$ '" >> ~/.bashrc   # le rendre permanent

Ajouter de la couleur

Syntaxe : \[\033[ATTRIBUT;COULEURm\]. Exemple : PS1='\[\033[32m\]\u@\h\[\033[0m\] ' affiche utilisateur@hôte en vert. Toujours réinitialiser avec \033[0m à la fin.

Commandes internes/externes, PATH & aide

D'où vient une commande, comment la retrouver, et comment obtenir de l'aide dessus.

Commandes internes vs externes

Une commande interne (builtin) fait partie du shell lui-même (ex : cd, echo). Une commande externe est un fichier binaire ou script séparé, cherché dans les dossiers de $PATH.

Bash
type echo     # "echo is a shell builtin" → interne
type bash     # "bash is /usr/bin/bash" → externe
which which   # affiche le chemin complet d'une commande externe
file fichier  # type d'un fichier : exécutable, texte ASCII/UTF-8…

La variable PATH

$PATH liste les dossiers où le shell cherche les commandes externes, séparés par :. C'est pourquoi un script dans le dossier courant doit être lancé avec ./script.sh (le dossier courant n'est pas dans PATH par défaut).

Bash
echo $PATH
chmod +x ma_commande.sh
./ma_commande.sh                    # fonctionne : chemin explicite
ma_commande.sh                      # échoue : "commande introuvable"
export PATH=$PATH:/root             # ajoute /root à la recherche (temporaire)

Obtenir de l'aide, historique & alias

CommandeUsage
help commandeaide d'une commande interne (ex : help echo)
commande --helpaide d'une commande externe
man commandemanuel complet ; /motif pour chercher, q pour quitter
whatis commanderésumé en une ligne du rôle d'une commande
history / !42historique / relancer la commande n°42
alias ll='ls -lrtha'créer un raccourci (permanent via ~/.bashrc)

Configurer le réseau

Sans réseau, pas de mises à jour ni d'installation de logiciels. On configure l'IP, la passerelle et le DNS.

IP statique — /etc/network/interfaces

On vérifie d'abord l'existant avec ip a (l'équivalent de ipconfig sous Windows) et ip r (table de routage), puis on édite le fichier des interfaces :

sudo nano /etc/network/interfaces
auto lo
iface lo inet loopback

auto ens33
iface ens33 inet static
    address 192.168.1.10
    netmask 255.255.255.0
    gateway 192.168.1.2
    dns-nameservers 8.8.8.8
Bash
sudo systemctl restart networking
ip a               # vérifier l'adresse
ping 8.8.8.8       # tester internet (Ctrl+C pour arrêter)

DNS — /etc/resolv.conf

Le DNS traduit www.google.fr en adresse IP. S'il manque, on ajoute nameserver 8.8.8.8 dans /etc/resolv.conf, puis on teste avec ping www.google.fr.

Diagnostic express

Si ping 8.8.8.8 fonctionne mais pas ping www.google.fr, le réseau est bon : c'est le DNS qui est en cause → vérifier /etc/resolv.conf.

Gérer les interfaces à chaud avec la commande ip

Le paquet iproute2 (commande ip) remplace l'ancien net-tools (ifconfig, non maintenu depuis 2009). Les changements faits avec ip sont temporaires : perdus au redémarrage si non écrits dans /etc/network/interfaces.

Bash
ip link show                              # liste les interfaces (up/down, MAC)
sudo ip link set eth0 down                # désactive l'interface
sudo ip addr add 192.168.76.10/24 dev eth1   # IP temporaire sur eth1
sudo ip route add default via 192.168.76.1 dev eth1   # passerelle par défaut

Résolution de noms : l'ordre de priorité

FichierRôle
/etc/nsswitch.confdéfinit l'ordre de résolution (ligne hosts: files dns → fichier local avant DNS)
/etc/hostsdictionnaire de résolution local (ex : 192.168.1.31 Facir)
/etc/resolv.confliste des serveurs DNS interrogés (nameserver ...)

Réseau & SSH au quotidien

Diagnostiquer le réseau et prendre la main à distance.

CommandeAction

nslookup domaine / dig domainetester la résolution DNS
curl -I url / wget urlen-têtes HTTP / télécharger un fichier

SSH : le terminal à distance, chiffré

Bash — côté serveur Debian
sudo apt install ssh
sudo systemctl enable ssh     # démarrage automatique au boot
systemctl status ssh          # vérifier que le service tourne
Depuis une autre machine (Windows, Mac, Linux)
ssh apprenant@192.168.1.10                # ouvrir une session distante
scp fichier apprenant@192.168.1.10:/home  # copier un fichier via SSH

Se connecter sans mot de passe (authentification par clé)

SSH combine deux types de cryptographie : asymétrique (clé publique/privée) pour établir la connexion de façon sûre, puis symétrique (plus rapide) pour le reste de la session. On génère une paire de clés côté client et on diffuse la clé publique sur le serveur :

Bash — côté client
ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa   # génère la paire (privée + .pub)
ssh-copy-id -i ~/.ssh/id_rsa.pub user@serveur # diffuse la clé publique

Permissions du dossier .ssh

Le dossier ~/.ssh doit être en 700 (accès au seul propriétaire) sinon SSH refuse de l'utiliser : chmod 700 ~/.ssh.

Transfert de fichiers

wget url / curl -O url téléchargent un fichier ; scp -r dossier user@hôte:/dest copie récursivement via SSH. Pour un transfert interactif façon explorateur, un client SFTP (FileZilla, WinSCP) se connecte sur le port 22 — le même service que SSH.

Le processus n°1

Au démarrage, Linux lance un premier processus d'ID 1 (init ou systemd). Tous les autres en sont les fils, petits-fils, etc. — visible avec ps -ef (colonne PPID = parent).

Bash
ps -ef | grep nom                   # filtrer par nom/utilisateur
ps -edf --sort=+pmem | tail -5      # les 5 processus qui consomment le plus de RAM
ps -edf --sort=+pcpu | tail -5      # les 5 qui consomment le plus de CPU

Raccourcis dans top

t bascule l'affichage CPU, m l'affichage RAM, h l'aide, q quitte. Le « load average » en haut indique le nombre de processus en attente de CPU sur 1/5/15 minutes.

Les services (systemd)

Un service est un programme qui tourne en permanence en arrière-plan (serveur web, SSH…). On les gère avec systemctl :

CommandeAction
sudo systemctl start/stop/restart servicedémarrer / arrêter / redémarrer
systemctl status servicevoir l'état
sudo systemctl enable/disable serviceactiver / désactiver au démarrage
journalctl -u service / journalctl -flogs d'un service / tous les logs en direct

À retenir

start agit maintenant, enable agit au prochain démarrage. Pour un service qui doit toujours tourner (comme SSH), il faut les deux.

Logs système & audit

Où regarder quand quelque chose s'est mal passé, ou pour savoir qui a fait quoi.

La plupart des processus envoient leurs traces au service rsyslog (config dans /etc/rsyslog.conf). Sur les systèmes récents (systemd), journalctl centralise aussi les logs.

Fichier / commandeContenu
/var/log/auth.logauthentifications, connexions SSH, élévations de privilèges (Debian)
/var/log/syslogjournal général du système
/var/log/kern.logmessages du noyau (détection matérielle, cartes réseau…)
/var/log/wtmphistorique des connexions/redémarrages (binaire, lire avec last)
journalctl -u service / -flogs d'un service précis / suivre en direct
Bash — qui a fait quoi ?
last                     # historique des connexions (lit /var/log/wtmp)
last | grep reboot       # uniquement les redémarrages du système
who                      # qui est connecté en ce moment
w                        # qui est connecté + ce qu'il fait + temps d'inactivité

Pourquoi wtmp est en binaire

Contrairement aux logs texte, /var/log/wtmp est stocké en binaire par le noyau : plus difficile à trafiquer pour effacer ses traces qu'un simple fichier texte.

Partitions, GRUB & réseaux de virtualisation

Des notions utiles dès l'installation ou quand on travaille avec des machines virtuelles (VMware, Hyper-V, VirtualBox).

Les partitions principales sous Debian

PartitionRôleAnalogie Windows
/ (racine)tout le système : binaires, config, logs. Si elle est pleine, le système peut devenir instable.C:\Windows + Program Files
/homedonnées des utilisateurs. La séparer permet de réinstaller le système sans perdre les données perso.C:\Users

Taille de swap recommandée

RAM ≤ 2 Go → swap = 2× la RAM · RAM 2-8 Go → swap = RAM · RAM > 8 Go → 4-8 Go suffisent souvent.

GRUB : le démarrage du système

GRUB (GRand Unified Bootloader) est le premier programme lancé après le BIOS/UEFI : il charge le noyau Linux. Équivalent du Windows Boot Manager (bootmgr). Il permet de choisir quel OS ou quelle version du noyau démarrer, et sa config est générée dans /boot/grub/grub.cfg (ne pas éditer directement) à partir de /etc/default/grub.

Bash
sudo update-grub              # régénère grub.cfg après modif de /etc/default/grub
sudo grub-install /dev/sda    # réinstalle GRUB (problème de boot)

Mode recovery

Si le système ne démarre plus normalement, GRUB permet de démarrer en mode « single »/« recovery » pour réparer, ou de revenir à un ancien noyau si une mise à jour a posé problème.

Modes réseau en virtualisation

ModeAccès InternetAccessible de l'extérieurCas d'usage
Host-only / Interne / Privénonnonlab isolé, VMs qui se parlent entre elles sans sortir sur le réseau réel
NAToui (via l'hôte)nonVM qui a juste besoin d'internet, sans être exposée

Sous Hyper-V

Ces 3 modes correspondent aux « commutateurs virtuels » : Externe (bridged), Interne (communique avec l'hôte mais pas le réseau physique) et Privé (isolé, même l'hôte ne communique pas avec les VMs).
