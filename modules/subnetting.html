Subnetting IPv4
Du binaire au VLSM, avec la méthode du calcul mental

01

Pourquoi découper un réseau

Une adresse IPv4 fait 32 bits, écrits en 4 octets décimaux (ex. 192.168.1.10). Elle se coupe en deux parties : une partie réseau (commune à tous les hôtes du segment) et une partie hôte (unique par machine). Le subnetting, c'est déplacer la frontière entre les deux.

À quoi ça sert concrètement

  • Segmenter — séparer serveurs, postes, Wi-Fi invité, VoIP… chacun son sous-réseau.
  • Sécuriser — un pare-feu ou des ACL filtrent entre sous-réseaux (cf. module CCNA).
  • Réduire le bruit — un domaine de broadcast plus petit = moins de trafic parasite.
  • Économiser les adresses — donner juste ce qu'il faut à chaque segment (VLSM).

Les classes, un vieux repère

Classe1er octetMasque « par défaut »Plages privées (RFC 1918)
A1 – 126/8 (255.0.0.0)10.0.0.0 – 10.255.255.255
B128 – 191/16 (255.255.0.0)172.16.0.0 – 172.31.255.255

Les classes ne servent plus qu'à situer une adresse : depuis le CIDR, le masque est libre et découplé de la classe.

À retenir

Une IPv4 = 32 bits = partie réseau + partie hôte. Subnetter, c'est allonger la partie réseau pour créer plusieurs sous-réseaux plus petits.

02

Le binaire sans douleur

Tout le subnetting tient dans un octet de 8 bits et ses puissances de 2. Apprends cette ligne par cœur, et 90 % du travail est fait.

La ligne magique (valeurs des bits d'un octet)

Bit12345678
Valeur1286432168421

Un octet = la somme des valeurs dont le bit vaut 1. Chaque bit à 1 « allume » sa valeur :

Conversion binaire → décimal
# 11000000  →  128 + 64                    = 192
# 10101000  →  128 +      32 +      8       = 168
# 11111111  →  128+64+32+16+8+4+2+1         = 255
# Donc 11000000.10101000.00000001.00001010 = 192.168.1.10

Décimal → binaire : la méthode « je soustrais »

Pour convertir 200 : je prends les valeurs de gauche à droite. 128 ≤ 200 → bit 1, reste 72. 64 ≤ 72 → bit 1, reste 8. 32,16 > 8 → bits 0. 8 ≤ 8 → bit 1, reste 0. Le reste à 0 → tout le reste est 0. Résultat : 11001000.

À retenir

128-64-32-16-8-4-2-1 : la seule chose à connaître par cœur. Un octet plein (11111111) = 255. Sait convertir dans les deux sens et tu tiens le subnetting.

03

Le masque de sous-réseau & le CIDR

Le masque dit où s'arrête le réseau et où commence l'hôte : les bits à 1 = réseau, les bits à 0 = hôte. La notation CIDR /n compte simplement le nombre de bits à 1.

Masque décimal ⇄ CIDR

Les deux formules qui gouvernent tout

Nombre d'hôtes

  • 2(bits hôte) − 2
  • Le « −2 » = adresse réseau + broadcast, non attribuables
  • Ex. /26 → 26 − 2 = 62 hôtes

Nombre de sous-réseaux

  • 2(bits empruntés)
  • Bits « empruntés » à la partie hôte pour agrandir le réseau
  • Ex. d'un /24 vers /26 → 2 bits → 4 sous-réseaux

À retenir

Masque = 1 pour le réseau, 0 pour l'hôte. Hôtes = 2h − 2 ; sous-réseaux = 2bits empruntés. La colonne « block size » (256 − dernier octet du masque) est ton meilleur ami au chapitre suivant.

04

Découper un réseau, étape par étape

Prenons un cas concret : on nous donne 192.168.10.0/24 et on veut 4 sous-réseaux. Voici la démarche complète, réutilisable à chaque fois.

  • 01
    Combien de bits emprunter ?4 sous-réseaux → 22 = 4, donc j'emprunte 2 bits. Le /24 devient /26 (24 + 2).
  • 02
    Nouveau masque & block size/26 = 255.255.255.192. Block size = 256 − 192 = 64. Les sous-réseaux avancent donc de 64 en 64.
  • 03
    Lister les sous-réseauxOn part de 0 et on ajoute le block size : .0, .64, .128, .192. Ce sont les 4 adresses réseau.
  • 04
    Broadcast & plage utilisableLe broadcast d'un sous-réseau = l'adresse juste avant le suivant. La plage utilisable = entre réseau+1 et broadcast−1.
Sous-réseauRéseau1ᵉʳ hôteDernier hôteBroadcast
#1192.168.10.0.1.62192.168.10.63
#2192.168.10.64.65.126192.168.10.127
#3192.168.10.128.129.190192.168.10.191
#4192.168.10.192.193.254192.168.10.255

À retenir

La recette : bits empruntés → nouveau masque → block size (256 − octet) → on avance de block size en block size. Le broadcast, c'est toujours « l'adresse juste avant le sous-réseau suivant ».

05

Le calcul mental rapide

La question type d'exam : « À quel sous-réseau appartient 172.16.20.170/27 ? » On répond en 15 secondes, sans binaire, grâce au block size.

  • 01
    Block size/27 → masque 255.255.255.224. Block size = 256 − 224 = 32.
  • 02
    Multiples du block sizeDans l'octet concerné (le 4ᵉ ici) : 0, 32, 64, 96, 128, 160, 192… Je cherche le multiple juste ≤ 170.
  • 03
    Encadrer l'adresse160 ≤ 170 < 192. Donc réseau = 172.16.20.160, broadcast = 172.16.20.191 (juste avant 192).
  • 04
    RéponseSous-réseau 172.16.20.160/27 · hôtes utilisables .161 → .190. L'adresse .170 est bien un hôte valide.

Le réflexe « block size »

256 − dernier octet non-nul du masque = block size. Les sous-réseaux tombent sur les multiples de ce nombre. C'est la clé pour tout faire de tête, sans jamais poser une conversion binaire.

À retenir

Pour situer une IP : trouve le block size, liste ses multiples, encadre l'adresse entre deux multiples. Le multiple du bas = le réseau, le suivant − 1 = le broadcast.

06

VLSM — des sous-réseaux sur mesure

Le VLSM (Variable Length Subnet Mask) utilise des masques différents selon les besoins. Au lieu de gaspiller un /24 entier pour une liaison à 2 hôtes, on donne à chaque segment juste ce qu'il lui faut. Règle d'or : servir les plus gros besoins d'abord.

Cas d'école — bloc 192.168.1.0/24

Besoins : Ventes 50 hôtes · IT 25 hôtes · Wi-Fi 10 hôtes · Liaison WAN 2 hôtes. On classe du plus grand au plus petit, puis on choisit le plus petit masque qui contient chaque besoin.

SegmentBesoinMasque choisiCapacitéBloc alloué
Ventes50/2662192.168.1.0 → .63
IT25/2730192.168.1.64 → .95
Wi-Fi10/2814192.168.1.96 → .111
Liaison WAN2/302192.168.1.112 → .115

Chaque bloc commence là où le précédent s'arrête. Il reste .116 → .255 de libre pour la croissance future : c'est tout l'intérêt du VLSM, zéro gaspillage.

À retenir

VLSM = un masque par besoin. Trie du plus grand au plus petit, prends le plus petit masque qui couvre le besoin (2h − 2 ≥ besoin), et empile les blocs sans trou. Une liaison point-à-point = toujours un /30.

07

Exercices corrigés

Entraîne-toi : cache la correction, réponds, puis vérifie. Refais-les jusqu'à répondre de tête en moins de 30 secondes.

Exercice 1 — situer une adresse

Question : pour 10.0.0.201/26, donne le réseau, le broadcast et la plage d'hôtes.

Correction

/26 → masque .192 → block size 64. Multiples : 0, 64, 128, 192, 256. 192 ≤ 201 < 256 → réseau 10.0.0.192, broadcast 10.0.0.255, hôtes .193 → .254.

Exercice 2 — combien de sous-réseaux & d'hôtes ?

Question : on découpe 172.16.0.0/16 en /24. Combien de sous-réseaux ? Combien d'hôtes chacun ?

Correction

Bits empruntés = 24 − 16 = 8 → 28 = 256 sous-réseaux. Bits hôte = 32 − 24 = 8 → 28 − 2 = 254 hôtes par sous-réseau.

Exercice 3 — VLSM express

Question : sur 192.168.50.0/24, alloue un segment de 100 hôtes, un de 50, puis une liaison /30.

Correction

100 → /25 (126 hôtes) : .0 → .127. 50 → /26 (62 hôtes) : .128 → .191. Liaison /30 : .192 → .195. Reste .196 → .255 disponible.
